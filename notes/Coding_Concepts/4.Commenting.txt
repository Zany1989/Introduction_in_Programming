4. Комментирование.

Объяснение концепции комментирования.

Когда разработчики работают над приложением, созданным кем-то другим, им необходимо знать некоторые подробности о коде,
чтобы иметь возможность его модифицировать. Комментарии — это способ объяснить, что делает код, и почему он это делает.
Это может быть полезно для других разработчиков, которые могут работать над кодом в будущем, или для вас, если вы
вернетесь к своему коду через некоторое время и не будете помнить, что он делает.

Комментарии — это способ добавления заметок, поясняющих то, что вы делаете. Чтобы компьютер понял, что то, что вы
пишете, не является кодом, вам нужно «закомментировать это», используя специальные символы для каждого языка
программирования. Например, в JavaScript вы можете использовать двойной слэш (//) для однострочных комментариев, а в
Python — символ решетки (#).

Таким образом, он проигнорирует эту часть, и программа будет работать правильно, но другие разработчики смогут понять,
что вы пытались сделать. Комментарии также могут быть полезны для объяснения сложных частей кода или для предоставления
дополнительной информации о том, как использовать определенные функции или классы.

Пример в JavaScript:
```javascript
// Эта функция принимает два числа и возвращает их сумму
function add(a, b) {
    return a + b; // Возвращаем результат сложения
}
```

Пример в Python:
```python
# Эта функция принимает два числа и возвращает их сумму
def add(a, b):
    return a + b  # Возвращаем результат сложения
```
