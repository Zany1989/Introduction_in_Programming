1. Git и GitHub.

Что такое Git и почему он полезен?

Git — это система контроля версий, которая позволяет разработчикам отслеживать изменения в коде и сотрудничать с другими.

Раньше решения заключались в сохранении версий на локальных жестких дисках, но это было неэффективно и не позволяло
легко сотрудничать с другими. Git позволяет разработчикам работать над кодом одновременно, не мешая друг другу, и
сохранять историю изменений.

Эта система легко настраивается и доступна в облаке. Git помогает отслеживать, над чем работают другие члены команды и
какая версия кода является последней. Это особенно полезно для больших проектов, где несколько разработчиков работают над
одним кодом.

GitHub — компания, которая взяла за основу процесс Git и создала собственный крупный сервис для сохранения кода. GitHub
позволяет разработчикам хранить свои проекты в облаке, делиться ими с другими и сотрудничать над ними.

«Форкинг» — модификация версии кода для создания собственной версии. Это позволяет разработчикам вносить изменения в код,
не затрагивая оригинальную версию.

«Пулл-реквест» — запрос на внесение изменений в оригинальную версию кода. Это позволяет разработчикам предлагать свои
изменения для рассмотрения и возможного включения в основную версию.
